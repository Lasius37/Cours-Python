    Principe général

    Lorsqu'un programme rencontre une erreur, Python s'arrête immédiatement. Ce comportement est à éviter : il faut faire en sorte que le programme puisse continuer. C'est le rôle des blocs try/except. 

    try:  # Python essaie de suivre ces instructions
    x = int(input("Nombre : "))
    print(10 / x)
except:  # En cas d'erreur dans le bloc try, il suivra ces instructions
    print("Il y a une erreur.")

    Types d'erreurs fréquents

    ValueError Conversion impossible int("abc")
    ZeroDivisionError Division par 0 37 / 0
    TypeError Mélange de types de variables incompatibles "abc" + 3
    KeyError Clé inexistante dans un dictionnaire moyenne["japonais"]
    IndexError Index trop grand dans une liste ou un tuple matieres[32]

    Gérer plusieurs types d'erreurs

    Dans le bloc try/except ci-dessous, il y a plusieurs causes d'erreur possible :

    • L'utilisateur ne saisit pas un nombre (ValueError).
    • L'utilisateur saisit 0 (ZeroDivisionError).

    Il est conseillé de gérer chaque type d'erreur indépendamment. On peut donc améliorer le code :

    try:  # Python essaie de suivre ces instructions
    x = int(input("Nombre : "))
    print(10 / x)
except ValueError:  # En cas de conversion impossible, il suivra ces instructions
    print("Vous devez saisir un nombre.")
except ZeroDivisionError:  # En cas d'erreur de division, il suivra ces instructions
    print("Il est impossible de diviser par 0.")
except:  # En cas d'erreur d'un autre type, il suivra ces instructions
    print("Erreur inconnue.")

    Erreurs fréquentes

    • Mauvais ordre dans les except : comportement anormal.
